# Урок 2.19. Хук `useId` (памятка)

## Что делает `useId`

`useId` генерирует стабильный уникальный id для конкретного экземпляра компонента.
Главная задача: связать элементы интерфейса через атрибуты `id`, `htmlFor`, `aria-*`.

Пример:

```jsx
const inputId = useId();

<label htmlFor={inputId}>Имя</label>
<input id={inputId} />
```

## Зачем это нужно

- Чтобы корректно работали подписи полей (`label` + `input`).
- Чтобы улучшить доступность (screen reader читает связи правильно).
- Чтобы избежать конфликтов id при рендере нескольких одинаковых компонентов.

## Как `useId` используется в твоем коде

## 1) Связь кнопки с формой вне `<form>`

Файл: `src/components/Form.jsx`

- `formId = useId()` используется как `id` формы.
- Кнопка с `form={formId}` отправляет форму, даже находясь вне тега `<form>`.

Это хороший практический кейс `useId`.

## 2) Доступные поля формы и сообщение об ошибке

Файл: `src/components/AccessibilityComponents.jsx`

- `nameId`, `emailId` связывают `label` и `input` через `htmlFor`/`id`.
- `errorId` используется в `aria-describedby`, чтобы оба поля ссылались на общее сообщение.

Это правильный и важный паттерн для accessibility.

## 3) Базовый префикс id для динамических форм

Файл: `src/components/AccessibleForms.jsx`

- `baseId = useId()` создаёт уникальный префикс для экземпляра компонента.
- Далее на его основе строятся id форм и полей.

Подход корректный: id не пересекаются между разными экземплярами компонента.

## 4) Использование в `App`

Файл: `src/components/App/App.jsx`

- `const id = useId()` и `const emailId = \`${useId()}-email\`` работают корректно.
- Для читаемости чаще удобнее вызывать `useId()` один раз на каждую сущность и отдельно формировать суффикс.

## Правила, которые стоит запомнить

- Вызывай `useId()` только на верхнем уровне компонента (как другие хуки).
- Используй `useId` для связей в DOM и a11y, а не для React `key`.
- Не вызывай `useId()` в циклах, условиях, вложенных функциях.
- Если нужно несколько связанных id, делай префикс:

```jsx
const baseId = useId();
const emailId = `${baseId}-email`;
const passwordId = `${baseId}-password`;
```

## Частые ошибки

- Использовать `useId` как `key` в списке.
- Генерировать id случайно (`Math.random`) для полей формы.
- Ломать правила хуков (вызов внутри `if`, `map`, обработчиков).

## Краткий вывод

В твоём проекте `useId` применяется по назначению: для связи формы, полей и ARIA-описаний.
Лучшие примеры: `Form.jsx` и `AccessibilityComponents.jsx`.
`AccessibleForms.jsx` хорошо показывает работу с динамическими структурами через общий `baseId`.
