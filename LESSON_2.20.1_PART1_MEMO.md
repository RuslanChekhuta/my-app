# Памятка: Урок 2.20.1 (часть 1/10)
## Самое крутое TODO-приложение (с голосовым вводом)

## 1) Что уже реализовано в вашем коде

На текущем этапе у вас собрана базовая версия TODO-приложения:
- добавление задачи через форму;
- удаление задачи;
- отметка выполнения (локально внутри `TodoItem`);
- переключение светлой/тёмной темы;
- сохранение темы в `localStorage`;
- Tailwind-стили и `dark`-вариант через `data-theme`.

Важно: **голосовой ввод пока не реализован**, но архитектура уже подходит как база для следующего шага.

## 2) Архитектура по файлам

- `src/App.jsx`
  - хранит список задач `todos`;
  - хранит текущую тему `theme`;
  - передаёт обработчики `onAdd`, `onDelete` в дочерние компоненты.

- `src/components/AddTodo.jsx`
  - контролируемый инпут (`text` в `useState`);
  - по `submit` валидирует `text.trim()` и вызывает `onAdd(text)`.

- `src/components/TodoItem.jsx`
  - локально хранит `isCompleted`;
  - умеет переключать статус задачи и удалять задачу.

- `src/components/ToggleTheme.jsx`
  - кнопка-тумблер для темы;
  - отображает текущую тему текстом.

- `src/helpers/getInitialTheme.js`
  - при старте выбирает тему из:
    1. `localStorage`,
    2. системной темы (`prefers-color-scheme`),
    3. времени суток.

- `src/helpers/toggleTheme.js`
  - переключает `light/dark`;
  - сохраняет новую тему в `localStorage`.

## 3) Как идет поток данных

1. Пользователь вводит текст в `AddTodo`.
2. `AddTodo` вызывает `onAdd` из `App`.
3. `App` обновляет `todos`, список перерисовывается.
4. `TodoItem` может удалить элемент через `onDelete`.
5. Тема меняется в `App`, а стили переключаются через `data-theme`.

## 4) Что стоит запомнить из части 1/10

- Поднимать состояние выше (`App`) полезно, если оно нужно нескольким компонентам.
- Контролируемые поля формы проще валидировать и очищать.
- Отдельные helper-функции (`getInitialTheme`, `toggleTheme`) делают код чище.
- UI-логику (кнопки, инпуты) лучше держать в компонентах, а данные — ближе к корню.

## 5) Технические замечания (полезно улучшить)

- В `onAdd` лучше использовать функциональное обновление:
  - сейчас: `setTodos([...todos, newTodo])`;
  - надёжнее: `setTodos((prev) => [...prev, newTodo])`.
  Это защищает от потенциальных гонок при быстрых обновлениях.

- `isCompleted` хранится внутри `TodoItem`, поэтому:
  - состояние выполнения не сохраняется между перезагрузками;
  - сложнее делать фильтры (выполненные/активные).
  Для роста приложения лучше хранить `completed` внутри каждого todo в `App`.

- `id: Date.now()` обычно работает, но теоретически может дать коллизию при очень быстрых действиях.

## 6) Подготовка к голосовому вводу (следующие части)

Минимальный план интеграции Web Speech API:

1. Проверка поддержки:
   - `window.SpeechRecognition || window.webkitSpeechRecognition`.

2. Добавление состояния:
   - `isListening`, `isSupported`, `speechError`.

3. Управление распознаванием:
   - старт/стоп по кнопке микрофона;
   - обработчик `onresult` для получения текста;
   - обработчик `onerror` для ошибок доступа/сети.

4. Интеграция с формой:
   - результат речи подставлять в `text` инпута (`AddTodo`),
     затем пользователь подтверждает добавление.

5. UX и безопасность:
   - подсветка активной записи;
   - явный индикатор «слушаю»;
   - корректное завершение `recognition` при размонтировании.
