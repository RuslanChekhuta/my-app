# Урок 2.16: хук `useTransition` (на основе вашего кода)

## Что у вас в проекте

Во всех примерах используется одна и та же связка:

```jsx
const [isPending, startTransition] = useTransition();
```

- `startTransition(...)` вы используете для "несрочных" `setState`.
- `isPending` показываете пользователю как индикатор загрузки (`"Загрузка..."`, `"Поиск..."`, `"Загрузка вкладки..."`).

## Пример 1: `BasicExample.jsx`

Файл: `src/components/BasicExample.jsx`

Что происходит:
1. По клику вызывается `startTransition`.
2. Внутри transition увеличивается `count`.
3. Пока обновление выполняется, вместо числа выводится `"Загрузка..."`.

Вывод: это минимальная демонстрация механики `useTransition` без тяжелых вычислений.

## Пример 2: `FilterExample.jsx`

Файл: `src/components/FilterExample.jsx`

Что происходит:
1. `query` обновляется сразу (`setQuery(value)`) это правильно, поле ввода остается отзывчивым.
2. Тяжелее часть (фильтрация 10 000 элементов) идет внутри `startTransition`.
3. Пока идет пересчет, показывается `"Поиск..."`.

Ключевая мысль на вашем коде: срочное состояние (`query`) отдельно, несрочное (`filteredData`) отдельно.

## Пример 3: `SlowRenderExample.jsx`

Файл: `src/components/SlowRenderExample.jsx`

Что происходит:
1. `input` обновляется сразу.
2. Создание большого списка (`ITEMS_COUNT = 10000`) вынесено в `startTransition`.
3. В UI показывается только часть (`VISIBLE_ITEMS = 200`), чтобы не перегружать DOM.

Ключевая мысль на вашем коде: `useTransition` управляет приоритетом обновления, а ограничение вывода (`slice`) уменьшает стоимость рендера.

## Пример 4: `TabExample.jsx` + `TransitionTabsBase.jsx`

Файлы:
- `src/components/TransitionTabsBase.jsx`
- `src/components/TabExample.jsx`

Что происходит:
1. В базовом компоненте `selectTab` делает `setTab(nextTab)` внутри `startTransition`.
2. При переключении вкладок React может показать `"Загрузка вкладки..."`.
3. Во вкладке `about` рендерится много элементов (`SlowPost`), поэтому переход помечен как несрочный.

Ключевая мысль: переключение тяжелой вкладки не блокирует срочные действия пользователя.

## Пример 5: `SyncTabExample.jsx`

Файл: `src/components/SyncTabExample.jsx`

Что важно:
1. Сам переход между вкладками тоже идет через `TransitionTabsBase` и `startTransition`.
2. Дополнительно в `SlowAboutTab` есть локальная асинхронность (`setTimeout` на 1000ms) через `useEffect`.

Ключевая мысль: `useTransition` и локальная загрузка данных внутри вкладки решают разные задачи и могут использоваться вместе.

## Что запомнить по вашему проекту

1. `useTransition` у вас применяется правильно там, где есть дорогие обновления (`filter`, генерация списка, тяжелые вкладки).
2. Срочные обновления (`input`, быстрые отклики UI) не обернуты в transition, и это правильно.
3. `isPending` используется по делу, как UI-сигнал, что несрочное обновление еще выполняется.
4. `useTransition` не "ускоряет" вычисления, он делает интерфейс более отзывчивым во время этих вычислений.
