СТРУКТУРА ПРОЕКТА

В соответствии с принципами масштабируемой архитектуры и правилами организации файловой структуры, необходимо подготовить следующую иерархию папок и файлов внутри директории `src`:

1.  **Папки верхнего уровня:**
    *   `src/components` — для переиспользуемых UI-элементов.
    *   `src/pages` — для компонентов-страниц.
    *   `src/assets` — для ресурсов, обрабатываемых сборщиком (изображения, шрифты).
    *   `src/utils` — для вспомогательных функций.
    *   `src/services` — для логики взаимодействия с бэкендом.
    *   `src/hooks` — для пользовательских хуков.
    *   `src/store` — для логики глобального стейта (резерв на будущее).

2.  **Структура компонента (Пример):**
    *   `src/components/Button/Button.jsx` — код компонента.
    *   `src/components/Button/Button.module.css` — стили компонента.
    *   `src/components/Button/index.js` — файл реэкспорта (опционально).

3.  **Структура страницы (Пример):**
    *   `src/pages/MainPage/MainPage.jsx`.

4.  **Файлы входа:**
    *   `src/main.jsx` (или `index.js` в зависимости от шаблона).
    *   `src/App.jsx`.

ЗАДАЧИ

#### Task 01
Создайте новый проект на Vite. Распакуйте базовый шаблон или инициализируйте новый (npm create vite@latest). Установите зависимости командой `npm i`. Запустите проект командой `npm run dev` и убедитесь, что стартовая страница отображается в браузере.
Изучите корневую структуру: папка `node_modules` (не редактируем), `public` (статика "как есть"), `package.json` (паспорт проекта) и `vite.config.js`.

#### Task 02
Приведите структуру папки `src` к профессиональному виду. Создайте внутри `src` следующие директории:
*   `components`
*   `pages`
*   `assets`
*   `utils`
*   `services`
*   `hooks`
*   `store`
Это обеспечит масштабируемость проекта и интуитивную навигацию по коду.

#### Task 03
Реализуйте принцип "один компонент — одна папка".
1.  В папке `src/components` создайте папку `Button`.
2.  Внутри `src/components/Button` создайте файл `Button.jsx` и файл стилей (например, `Button.css` или `Button.module.css`).
3.  Напишите простейший функциональный компонент кнопки в `Button.jsx`, который возвращает HTML-элемент `<button>Click me</button>`, и экспортируйте его.

#### Task 04
Создайте структуру для страницы.
1.  В папке `src/pages` создайте папку `MainPage`.
2.  Внутри создайте файл `MainPage.jsx`.
3.  В `MainPage.jsx` создайте компонент, который возвращает заголовок `<h1>Main Page</h1>` и импортированный компонент `Button`, созданный в предыдущей задаче.
4.  Отредактируйте корневой файл `App.jsx`: импортируйте туда `MainPage` и выведите его внутри возвращаемой разметки. Убедитесь, что в браузере отображается заголовок страницы и кнопка.

#### Task 05
Настройте Git для чувствительности к регистру имен файлов. Это критически важно для предотвращения ошибок при деплое на Linux-серверы, если вы решите переименовать файл (например, `MyName.jsx` в `myname.jsx`).
Откройте терминал в папке проекта и выполните команду:
`git config core.ignorecase false`
Теперь Git будет отслеживать изменения регистра как полноценное переименование файлов.
