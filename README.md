Task 02
• Целевой файл: src/components/JsxRules.jsx
• Действие: Создайте компонент, демонстрирующий правила JSX.
1. Оберните всё содержимое в React Fragment (`<>...</>`), чтобы не создавать лишний div в DOM.
2. Внутри фрагмента создайте `div` с классом `container` (используйте правильный атрибут JSX для классов).
3. Внутри div добавьте заголовок `h2` с текстом "Правила JSX".
4. Добавьте параграф с текстом "Каждый тег должен быть закрыт".
5. Добавьте `input` типа `text` (обязательно закройте тег!).
6. Добавьте `label` для этого инпута с текстом "Введите текст" (используйте правильный атрибут JSX для привязки к id инпута, вместо `for`).
• CSS-подсказка: Не требуется (классы уже описаны в Task 01).
• Проверка: Код: убедитесь, что использованы `className`, `htmlFor`, `< />` для input и Fragment `<>`.

Task 03
• Целевой файл: src/components/ExpressionPractice.jsx
• Действие: Пропрактикуйте встраивание выражений `{}`.
1. Внутри компонента (до return) создайте константы: `name` (строка), `age` (число) и `imgUrl` (строка с ссылкой на любую картинку, например, 'https://placehold.co/100').
2. В `return` верните `div`.
3. Внутри div выведите текст: "Привет, {name}!".
4. Выведите выражение: "В следующем году вам будет {age + 1}".
5. Выведите тег `img`, используя переменную `imgUrl` в атрибуте `src` и строку "avatar" в атрибуте `alt`.
• CSS-подсказка: Не требуется.
• Проверка: В браузере отображается имя, вычисленный возраст и картинка.

Task 04
• Целевой файл: src/components/StyleCard.jsx
• Действие: Реализуйте инлайн-стили (inline styles).
1. Создайте `div` с классом `card`.
2. Внутри создайте `h3` с текстом "Инлайн стили".
3. Добавьте `div`, у которого атрибут `style` задает:
   - Фон (background-color) светло-зеленый (`#e0f7fa`).
   - Закругление углов (border-radius) 10px.
   - Цвет текста темно-зеленый.
   - Текст внутри: "Я стилизован через объект!".
4. Убедитесь, что свойства CSS записаны в camelCase (например, `backgroundColor`).
• CSS-подсказка: Не требуется (используем атрибут style).
• Проверка: Код: `style={{ backgroundColor: '#e0f7fa', borderRadius: '10px' }}`.

Task 05
• Целевой файл: src/components/EmailValidator.jsx
• Действие: Реализуйте условный рендеринг.
1. Внутри компонента создайте константу `email`. Присвойте ей значение (пустую строку или валидный email) для теста.
2. В `return` создайте `div`.
3. Используя тернарный оператор или оператор `&&`, реализуйте логику:
   - Если `email` не пустой — вывести `h4` с текстом "Email: {email}".
   - Если `email` пустой — вывести `h4` с текстом "Email не указан", окрашенный в красный цвет (через style).
• CSS-подсказка: Не требуется.
• Проверка: Изменяя значение переменной `email` в коде, в браузере меняется надпись.
