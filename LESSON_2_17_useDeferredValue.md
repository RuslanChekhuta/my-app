# Урок 2.17. Хук `useDeferredValue`

## Что делает `useDeferredValue`
`useDeferredValue(value)` возвращает "отложенную" версию значения.

Это значит:
- срочный UI (например, ввод в `input`) обновляется сразу;
- тяжелая часть интерфейса получает значение позже, когда у React есть ресурс;
- интерфейс ощущается более отзывчивым.

## Разбор по вашему коду
### 1) Источник срочного значения
В `src/components/App/App.jsx`:
- `query` обновляется мгновенно при вводе;
- `const deferredQuery = useDeferredValue(query)` создает отложенную копию;
- в тяжелый список уходит именно `deferredQuery`.

### 2) Тяжелый рендер
В `src/components/SearchList.jsx`:
- рендерится 50 `SearchItem`;
- каждый `SearchItem` специально "тормозит" на ~10ms через цикл;
- суммарно получаем заметную нагрузку.

`useDeferredValue` в таком сценарии снижает лаги ввода: поле ввода живет на высоком приоритете, список может догонять.

## Почему это работает
- `query` меняется на каждом нажатии клавиши.
- `deferredQuery` может отставать (это нормально).
- пока тяжелый список пересчитывается для старого значения, пользователь продолжает печатать без "залипания" input.

## Как понять, что значение отложено
Полезный признак:

```jsx
const isStale = query !== deferredQuery;
```

Если `isStale === true`, можно показать индикатор "Обновление списка...".

## `useDeferredValue` vs `useTransition`
`useDeferredValue`:
- откладывает производное значение;
- удобно, когда уже есть состояние (`query`) и нужно передать его в тяжелый UI.

`useTransition`:
- откладывает конкретное обновление состояния внутри `startTransition`;
- дает `isPending` и больше контроля над тем, какие апдейты считать несрочными.

В вашем проекте:
- `App.jsx` демонстрирует `useDeferredValue`;
- `SearchItemWithTransition.jsx` демонстрирует похожую задачу через `useTransition`.

## Важный момент по текущему запуску
Сейчас в `src/main.jsx` рендерится `SearchItemWithTransition`, а `App` закомментирован.

Если хотите увидеть именно пример `useDeferredValue`, в `main.jsx` нужно временно рендерить `App`.

## Когда использовать `useDeferredValue`
- тяжелая фильтрация/списки/таблицы;
- live-поиск с большим количеством элементов;
- случаи, когда при печати заметны фризы из-за дорогого рендера ниже по дереву.

## Частые ошибки
- передавать в тяжелый компонент не `deferredQuery`, а исходный `query`;
- ожидать, что `deferredQuery` обновляется мгновенно (он специально может запаздывать);
- применять хук там, где нет реальной тяжелой работы.
