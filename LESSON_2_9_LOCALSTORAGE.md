# Урок 2.9. Кастомный хук для работы с `localStorage`

## Зачем нужен кастомный хук
`localStorage` хранит данные между перезагрузками страницы, но работать с ним напрямую в каждом компоненте неудобно.

Кастомный хук `useLocalStorage`:
- убирает дублирование кода;
- дает единый API: получить, записать, удалить значение;
- держит состояние React и `localStorage` синхронизированными;
- поддерживает синхронизацию между вкладками.

## Что делает ваш хук
Файл: `src/hooks/useLocalStorage.js`

Хук принимает:
- `key` - ключ в `localStorage`;
- `initialValue` - значение по умолчанию.

И возвращает:
- `storedValue` - текущее значение;
- `setValue(value)` - запись нового значения;
- `removeValue()` - удаление значения и сброс к `initialValue`.

## Пошаговая логика
1. При первом рендере вызывается `getStoredValue()`:
   - пытается прочитать `localStorage.getItem(key)`;
   - если значение есть, делает `JSON.parse`;
   - если нет или ошибка - возвращает `initialValue`.
2. Значение кладется в `useState(getStoredValue)`.
3. `setValue`:
   - сериализует значение через `JSON.stringify`;
   - сохраняет в `localStorage`;
   - обновляет React-состояние.
4. `removeValue`:
   - удаляет ключ из `localStorage`;
   - ставит `initialValue` в состояние.
5. `useEffect` с `storage`:
   - слушает изменения в других вкладках;
   - если изменился нужный `key`, обновляет состояние.

## Как используется в компонентах
### `App.jsx`
```jsx
const [name, handleSetName, handleRemoveName] = useLocalStorage("name", "Гость");
```
- поле ввода меняет имя и сразу сохраняет его;
- кнопка "Очистить имя" удаляет ключ и возвращает `"Гость"`.

### `Setting.jsx`
```jsx
const [theme, setTheme] = useLocalStorage("theme", "light");
const [lang, setLang] = useLocalStorage("lang", "ru");
```
- выбранные тема и язык сохраняются и восстанавливаются после перезагрузки.

## Плюсы вашей реализации
- Есть `try/catch` на чтение, запись и удаление.
- Есть поддержка `JSON`, значит можно хранить не только строки.
- Есть синхронизация между вкладками.
- Возврат в `initialValue` при удалении и ошибке парсинга.

## Важные нюансы
- `setValue` сейчас принимает только готовое значение, а не функцию-апдейтер.
  Пример отсутствующего сценария: `setValue(prev => prev + 1)`.
- Событие `storage` не срабатывает в той же вкладке, где вызвали `setItem`, поэтому локальное `setStoredValue` в `setValue` обязательно и у вас оно есть.
- При серверном рендеринге нужен дополнительный guard на `window/localStorage`.

## Краткий шаблон для повторения
```jsx
const [value, setValue, removeValue] = useLocalStorage("myKey", defaultValue);
```
- `value` - читаем;
- `setValue(newValue)` - сохраняем;
- `removeValue()` - очищаем.

## Итог
Ваш `useLocalStorage` уже закрывает типичный production-сценарий для клиентского React-приложения: хранение настроек, восстановление состояния после перезагрузки и синхронизация между вкладками.
